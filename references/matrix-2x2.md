# Matrix 2×2 / 战略矩阵 — 设计与数据规范

> 适用:`matrix-2x2.html` 模板 — 麦肯锡 / BCG 风的 2 维定位图,4 象限 + N 气泡。
>
> 这是咨询报告里最高频的页型,**一张图说清 N 个对象的相对位置 + 战略含义**。
> 适合:竞品定位、业务段优先级、客户分层、项目排序。

## 一、什么场景适合 2×2 矩阵?

✓ 你有 **N 个对象**(5-12 个最舒服,< 4 太空旷,> 15 太拥挤);
✓ 每个对象都能用**两个连续维度**度量;
✓ 你想让读者**一眼看出"谁是谁"**(分类,而不是排序);
✓ 这两个维度可以**自然交叉**形成 4 个有商业含义的象限;

✗ 不适合:维度多于 2 个(用热力图 / 雷达图);
✗ 不适合:数据本身是离散类别(用堆叠条 / 桑基图);
✗ 不适合:对象只有 2-3 个(直接横条对比就行)。

## 二、6 套经典轴组合(按场景选)

| # | 横轴 | 纵轴 | 气泡大小 | 适用 |
|---|---|---|---|---|
| 1 | 营收增速 YoY | 估值倍数 P/ARR | 绝对估值 | **竞品估值** (OpenAI deck 用) |
| 2 | 相对市场份额 | 市场增长率 | 业务规模 | **BCG 矩阵**(经典) — 业务段优先级 |
| 3 | 用户规模 | ARPU | 单段毛利 | **用户分层** — 谁是高价值客户 |
| 4 | 投入(人 / $) | 业务影响 | 项目数 | **项目优先级** — 收益矩阵 |
| 5 | 技术成熟度 | 商业化成熟度 | 投入 | **新业务孵化**(Gartner Hype 风) |
| 6 | 产品 NPS | 增长率 | MAU | **产品健康度** — 用户体验 vs 增长 |

**轴选择原则:**
- 两轴**必须独立**(不能强相关),否则对象会全部聚集在对角线一条带上
- 两轴都要**连续可量化**;若一轴是类别(如「公司类型」),改用其他页型
- 高数值方向用**正向语义**(增长 / 收益 / 规模),避免"高=坏"反直觉

## 三、数据规范(关键!从源头规范)

### 3.1 数据格式

每个对象一行,字段:

```json
{
  "name": "OpenAI",
  "x": 400,          // 横轴原始值 (例:YoY 增速 %)
  "y": 40,           // 纵轴原始值 (例:P/ARR 倍数)
  "size": 10000,     // 气泡大小 (例:估值 $亿)
  "color": "accent", // 配色(accent / accent2 / accent3 / warn / danger / 自定义)
  "meta": "$1万亿 · 40× · +400%",   // 标签下行小字
  "isRef": false     // true → 虚化(参考对象,无完整数据)
}
```

### 3.2 归一化公式(动手前先算)

矩阵内部用百分比定位,**必须先把 (x, y) 映射到 0-100%**:

```
left% = ((x - x_min) / (x_max - x_min)) * 100
top%  = 100 - ((y - y_min) / (y_max - y_min)) * 100   ← 注意 y 反向(CSS 顶部 0)
```

**实操建议:**
- 选 x_min / x_max 时**留 10-15% 缓冲**(避免对象贴边):
  - 如真实增速范围 50-450%,把 x_min=0, x_max=500 设宽一点
- 极值对象(如 xAI P/ARR 130× 远超其他)有两种处理:
  - 截断顶部(top:6%) + 在 b-meta 注明真实值
  - 用 **log 归一**:y_norm = log10(y) / log10(y_max)
- **大小气泡分级**用 `data-size="huge|big|mid|sm"` **离散映射**,不要连续:
  - huge ≥ $5000 亿 / big ≥ $1500 亿 / mid ≥ $300 亿 / sm < $300 亿
  - 连续映射会产生 18.7px 这种奇葩值,视觉上反而乱

### 3.3 标签防遮挡规则

- 默认气泡标签在**下方**(`flex-direction: column`)
- 若气泡在象限**上半部**(top < 30%),加 `b-label-up` 类把标签放上方
- 若两个气泡水平距离 < 10%,**两个都加** `b-label-up` 错开
- 象限标题永远固定在象限角落(z-index: 1),气泡 z-index: 4,标签 z-index: 5

## 四、内容规范(让矩阵有"咨询感")

### 4.1 4 象限命名(别用「象限 1/2/3/4」)

每个象限应有**带感情色彩的战略名**:

| 位置 | 高估值轴示例 | BCG 示例 | 项目优先级示例 |
|---|---|---|---|
| 右上(高 x · 高 y) | ★ 明星象限 | 明星业务 | 必做项目 |
| 左上(低 x · 高 y) | ⚠ 高估值陷阱 | 现金牛 | 战略储备 |
| 右下(高 x · 低 y) | 🚀 黑马象限 | 问题儿童 | 快赢项目 |
| 左下(低 x · 低 y) | 🐂 价值现金牛 | 瘦狗 | 砍掉 |

### 4.2 右侧 3 条战略洞察(必须有,不要省)

矩阵本身只展示「在哪」,**洞察告诉读者「所以呢」**:

```
01 {对象 A} 的位置含义 → 论据 → **可执行结论**
02 {对象 B} 的对比含义 → 论据 → **可执行结论**
03 整体格局含义        → 论据 → **可执行结论**
```

底部 `ms-foot` 一句话总结:**让读者带走一个判断**,而非整版细节。

## 五、踩坑黑名单

1. **绝不用 ECharts scatter 做这个**。
   ECharts 的 scatter 没法做 4 象限着色 + 居中标签 + 标签换行 + 象限文字定位 — 强行做要写一堆 graphic 配置,反而比纯 CSS 痛苦 10 倍。
   **结论:纯 CSS + `position:absolute` + `transform:translate(-50%,-50%)` 最干净**。
2. **绝不让对象气泡贴边**。x 或 y 在 0-5% / 95-100% 范围 → 标签会被裁;放宽数据范围或截断处理。
3. **绝不让标签重叠**。出现 → 加 `b-label-up` 错开,或调整其中一个 x 值 ±2%。
4. **绝不省 4 象限战略名**。「左上/右下」是位置描述,「明星象限/黑马象限」才是商业判断 — 没有这一步,矩阵就是个普通散点图。
5. **绝不在矩阵里塞 > 12 个气泡**。太多会变成"星空图",失去定位价值 — 取头部 7-10 个,长尾汇成「其他 (N 家)」一个虚化气泡。
6. **绝不忽略「ref 参考对象」**。某对象数据不完整或不是同类(如 OpenAI deck 里 Gemini 是大厂内部业务,没独立估值),用 `.ref` 虚化显示 → 告诉读者"我知道它在但不直接对比"。
