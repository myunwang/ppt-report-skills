# 设计系统：信息层次的硬规则

**核心原则**：每段文字必须能回答"我是哪一级"。读者扫页面时，只看字号/字重/颜色就能理出主次。

## 8 级文字层次

| 级别 | 用途 | CSS 变量 | 默认值 | 字重 | 颜色 |
|---|---|---|---|---|---|
| L0 | 顶部小标签（"月度总览"） | `--fs-label` | 13px | 600 | `var(--accent2)` 大写 + 字距 0.18em |
| L1 | 页面大标题 | `--fs-title` | 36px | 800 | `var(--text)` |
| L2 | 副标题 / 一句结论（含关键数字） | `--fs-subtitle` | 18px | 500 | `var(--text2)`，关键数字 `<strong>` `var(--accent)` |
| L3 | 区块/栏目标题 | `--fs-h2` | 18px | 700 | `var(--text)` |
| L4 | 卡片标题、metric 大数字 | `--fs-h3` | 15px | 600 | `var(--text)` |
| L5 | 正文、metric 名 | `--fs-body` | 13px | 400-500 | `var(--text2)` |
| L6 | 元信息、注释、card-conclusion | `--fs-caption` | 12px | 400 | `var(--text3)` |
| L7 | 辅助小字、tag、chart 注脚 | `--fs-mini` | 11px | 400-600 | `var(--text3)` |

> 字号在 1600×900 设计稿坐标系内。整页随窗口缩放，**不要再单独缩放字号**。

## 配色（语义先于审美）

主题变量统一前缀 `--accent*`、`--text*`，**所有页面和组件都用变量**，方便切主题。

| 变量 | 默认 | 含义 |
|---|---|---|
| `--bg` | `#f4f6fb` | 整页背景 |
| `--surface` | `#ffffff` | 卡片/面板底色 |
| `--surface2` | `#f1f4fa` | 嵌套卡片/输入框等次级面 |
| `--border` | `rgba(15,23,42,0.08)` | 描边 |
| `--accent` | `#2563eb` | 主品牌色（蓝）— 标题高亮、按钮、关键数字 |
| `--accent2` | `#0891b2` | 辅助色（青）— 次级强调 |
| `--accent3` | `#059669` | 正向 / 上涨 / 完成（绿）|
| `--warn` | `#d97706` | 注意 / 中性偏弱（橙）|
| `--danger` | `#dc2626` | 风险 / 下降（红）|
| `--text` | `#0f172a` | 主文字 |
| `--text2` | `#475569` | 次级文字 |
| `--text3` | `#94a3b8` | 弱化文字 |

## 增减语义色（强制规则）

数据汇报最常被滥用的就是颜色。**约定**：

- **绿（`--accent3`）** = 正向变化 / 完成 / 实验组优于对照组（**注意**：与 `data-report` skill 的"绿=下降"是相反约定，本 skill 用"绿=正向"，因为业务里 KPI 涨了通常是好事）
- **红（`--danger`）** = 负向变化 / 失败 / 风险
- **橙（`--warn`）** = 中性 / 不及预期 / 需要关注但不是 P0

JSX/HTML 里在结论文字内：

```html
<span class="pos">+30.5%</span>     <!-- 绿，正向 -->
<span class="neg">-2.7%</span>      <!-- 红，负向 -->
<span class="warn">仅 2.8%</span>   <!-- 橙，需要关注 -->
<strong>关键数字</strong>           <!-- 蓝，主品牌色高亮 -->
```

## 间距系统（4 的倍数）

只用以下间距（px，1600×900 设计稿）：

| token | px | 用途 |
|---|---|---|
| xs | 4 | 同行内极紧贴元素 |
| sm | 8 | tag / dot 与标签 |
| md | 12 | 卡片内行距 |
| lg | 16 | 区块间距、grid gap |
| xl | 20-24 | 卡片 padding |
| 2xl | 32-40 | 顶部留白、大区块间距 |
| 3xl | 48-64 | 页面外边距（slide padding 默认 48 64） |

```css
.slide { padding: 48px 64px; }      /* 顶部 48 / 左右 64 */
.kpi-grid { gap: 16px; }             /* lg */
.card { padding: 20px 24px; }        /* xl */
```

## 圆角

- 小元素（tag、dot 包围）：4px
- 卡片：10–12px
- 大面板：16px
- 全圆 / 胶囊：100px

## 阴影（柔和）

```css
--shadow-card:    0 1px 3px rgba(15,23,42,0.04), 0 1px 2px rgba(15,23,42,0.03);
--shadow-card-lg: 0 4px 12px rgba(15,23,42,0.06), 0 2px 4px rgba(15,23,42,0.04);
```

只在浅色主题用阴影。深色主题用 `border` + `--surface2` 区分层次。

## 强制顶部三件套

每页**必须**有：

```html
<div class="slide-label">月度总览</div>   <!-- L0 标签 -->
<div class="slide-title">4月工作交付概览</div>   <!-- L1 标题 -->
<div class="slide-subtitle">
  策略需求 <strong>5 个</strong>全部上线 ... — 月度核心交付完整
</div>   <!-- L2 一句结论，必须含关键数字 -->
```

`slide-subtitle` 是这一页的"中心结论"。读者**只读副标题**就能知道页面要点。

## 字重对照

字重不是装饰，是层级。**同页不要超过 4 种字重**：

- 900 → 大数字（KPI、col-num 那种）
- 700-800 → 大标题、列标题、强调
- 600 → 卡片标题、tag、status
- 400-500 → 正文、metric 名

## 反模式（禁止）

- ❌ 同一页里出现 2 种以上字号系统外的临时尺寸（如 `font-size: 14px`，应该用 `var(--fs-body)`）
- ❌ 用斜体强调（中文不认）
- ❌ 用下划线强调（看起来像超链接）
- ❌ 用 `text-shadow` 装饰
- ❌ 红绿色弱不友好的高饱和原色（`#ff0000`、`#00ff00`）
- ❌ 数字不用等宽字体（导致跳动）—— 数字必须 `font-family: var(--mono)`
