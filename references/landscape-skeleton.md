# Landscape 骨架选择 — 反八股强制流程

> 做"产业全景图 / 行业图谱 / 公司业务全景"类页面前的**骨架选择强制流程**。
> 本文档是为了治"看到全景图就套 landscape-map 三层 tier 八股"的毛病 —— 那个模板只
> 适合软件 / AI 行业,套到实体产业会被一眼识破"参考案例呆板"。
>
> **核心问题:** 不同行业的产业逻辑根本不一样。AI 是分层栈、SpaceX 是价值流、银行
> 是客户矩阵、Pfizer 是管线时间轴。
>
> **绝不允许** 用 landscape-map「应用层 / 通用技术 / 底层供应链」三层 tier 模板**套
> 所有行业** —— 那只是 AI / SaaS 类软件分层栈的专用骨架。

## 一、动手前先做骨架决策(必做)

**先回答这 3 个问题,再选骨架:**

1. **这个行业的核心驱动逻辑是什么?**
   - 软件能力分层(每层依赖下层) → A
   - 物理资产流转 / 价值流(原料→生产→流通→服务) → B
   - 客户分层 / 渠道矩阵(谁付钱 + 怎么触达) → C
   - 研发管线 / 时间轴 / 阶段推进 → D
   - 生态合作伙伴 / 联盟网络 → E

2. **核心要素是"分类 + 层级"还是"流程 + 节点"?**
   - 分类层级 → A / C 倾向
   - 流程节点 → B / D 倾向

3. **用户最想看到什么?**
   - 谁在每层 / 各分类有哪些玩家 → A / C
   - 价值如何被创造和分配 → B
   - 业务进展到哪一步 / 未来催化路径 → D
   - 谁和谁结盟、谁是核心节点 → E

## 二、5 种骨架与适用场景

### A · 分层栈(landscape-map 三层 tier)
**适用:** AI、SaaS、云计算、互联网产品、半导体设计栈
**结构:** 应用层(5 卡) → 通用技术(4 卡) → 底层供应链(2 嵌套卡)
**逻辑:** 每层是下层的能力消费方,层与层之间是"使用-被使用"关系
**模板:** `assets/slides-templates/landscape-map.html`
**配套自检:** 必读 `references/landscape-qa.md` 的 10 条踩坑黑名单 + 数据规范 + 渲染自检

### B · 价值流横轴(value-chain flow)
**适用:** 商业航天(SpaceX)、能源石化、汽车制造、消费品 / 食品、零售
**结构:** 5-7 段横向流程节点(▶ 箭头串联),每段含「角色 + 关键数字 + 自营 vs 外采标识」
**附加层:** 中部加客户矩阵(三类客户 + 占比),底部加上下游供应链 + 竞争对手双栏
**逻辑:** 资产沿价值链流动,各段毛利 / 自营度差异巨大
**参考实现:** `~/personal/spacex-ipo-deck/src/slides/slide-2.html`(基于本仓库 skill 做的)

### C · 客户矩阵 + 产品 × 渠道(customer-matrix)
**适用:** 银行 / 保险 / 券商、咨询、广告营销、B2B SaaS 销售视角
**结构:** 纵轴客户分层(零售 C 端 / 对公 B 端 / 同业机构),横轴产品线或渠道,交叉格填 KPI
**逻辑:** 谁付钱 + 用什么产品触达 + 这格利润 / 规模如何
**关键:** 每格放 3-4 个代表客户 + 一个核心指标(ARPU / GTV / 渗透率)

### D · 研发管线 / 时间轴(pipeline / roadmap)
**适用:** 生物医药、新药研发、芯片 IDM 路线、新能源车企产品规划、学术研究方向
**结构:** 横轴时间(已上市 / III 期 / II 期 / I 期 / 临床前)或代际(Gen1 / 2 / 3),纵轴按适应症或产品线
**逻辑:** 业务核心是"未来兑现的预期",每个节点都有里程碑日期 + 概率 + 价值量
**关键:** 配色按"已验证 → 高概率 → 中概率 → 高风险"梯度

### E · 生态网络(ecosystem-network)
**适用:** 平台型公司(微信 / 阿里 / Salesforce)、开源生态、行业联盟、标准组织
**结构:** 中心节点(平台 / 标准)+ 放射状外围(开发者 / 合作方 / 上下游),用线连接
**逻辑:** 价值在网络效应中创造,中心和边缘的关系决定护城河
**关键:** 用「关系类型」着色(投资 / 战略合作 / 技术授权 / 客户)

## 三、骨架选定后的强制开工流程

1. **写下选骨架的理由**(1-2 句),作为 HTML 注释放在 slide 顶端 ——
   ```html
   <!-- 骨架:B 价值流横轴
        理由:SpaceX 是重资产实体产业,核心逻辑是「制造→发射→在轨→地面→变现」的价值流转,
        各段自营度 / 毛利 / 自有壁垒差异巨大,套三层栈无法体现 -->
   ```
2. **数据先于设计:** 列出该骨架每个区域要填的字段(节点名 / 关键数字 / 代表实体),再写 HTML/CSS。
3. **chip 排版规范不变** —— 无论选哪个骨架,chip 字号固定 / 数据短词化 / logo Clearbit
   兜底,均按 `references/landscape-qa.md` 执行。
4. **首版生成后必须做"骨架适配性自检":**
   - 这个骨架能不能体现该行业的独特卖点?
   - 把数据换成另一个同类公司,这个图还成立吗?(成立 → 太通用,需更聚焦;不成立 → 骨架够特化)
   - 用户如果是行业专家,他第一眼会觉得"这就是我们这行的图"还是"模板套来的"?

## 四、常见反面案例(每次都该警惕)

| 错误套法 | 实际行业 | 应该选 |
|---|---|---|
| SpaceX 套「应用 / 通用技术 / 底层供应链」 | 商业航天(实体产业) | **B 价值流横轴** |
| 招商银行套「应用场景 / 通用技术 / 数据基础」 | 银行(客户驱动) | **C 客户矩阵** |
| 恒瑞医药套「治疗领域 / 研发平台 / 原料药」 | 创新药(管线驱动) | **D 研发管线** |
| 微信套「应用 / 中台 / 基础设施」 | 平台生态(网络效应) | **E 生态网络** |
| AI 公司套「价值流横轴」 | 通用 AI(分层栈) | **A 三层 tier** ✓ |

## 五、一句话

**先问"这行业靠什么驱动",再画图。不要看到"全景图"就摸 landscape-map 模板 —— 那只是 5 种骨架中的 1 种。**
