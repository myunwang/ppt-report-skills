# 汇报 PPT 经典布局法则

设计系统管"字怎么写、颜色怎么用"；本文管"一页地里东西怎么摆"。

## 0. 顶层心法：每页一个故事

**一页 = 一个结论 + 支撑证据**。读者扫页面的顺序应该是：

1. 看 `slide-title` 知道**主题**（3 秒）
2. 看 `slide-subtitle` 知道**结论**（5 秒）
3. 看主体的 1~3 个证据**确认结论**（15 秒）
4. （可选）看脚注 / card-conclusion **挖细节**

如果你的读者扫完前两步就能合上文档去开会，这页就赢了。**每页只回答一个问题**，问题写不出来 = 这页不该存在。

## 1. 金字塔结构（每页内部）

每页是一座倒金字塔：

```
┌─────────────────────────────────────┐
│ LABEL（小标签）                       │  ← 上下文锚点
│ TITLE 主题（最大字号）                │  ← 主题
│ SUBTITLE 结论（中等字号、含关键数字） │  ← 结论 ★★★
├─────────────────────────────────────┤
│                                     │
│      支撑证据（图表 / 表格 / 卡片）  │  ← 证据
│                                     │
├─────────────────────────────────────┤
│ FOOTER 注脚 / 数据源 / 备注（小）    │  ← 元信息
└─────────────────────────────────────┘
```

**比例参考**：标题区占页面高度 15-20%，主体 70-75%，footer ≤10%。在 1600×900 设计稿里 ≈ 标题 150-180px / 主体 630-680px / footer 不强制。

## 2. F 型 / Z 型阅读路径

读者眼睛进入页面后的路径，**用版面引导**而不是听天由命：

- **F 型**：信息密集型页面（多卡片、多表格）—— 视线从左上 → 横扫 → 下移再横扫。重要信息靠左、靠上。
- **Z 型**：观点型页面（少元素、有结论）—— 视线从左上 → 右上 → 左下 → 右下。CTA / 关键数字放右下"落点"。

**应用**：
- N 卡片 KPI 行，最重要的指标放**最左**
- 双对照页（A vs B），**结论强的一方放左**（读者先看到正向数据，更愿意继续读）
- 反例：把"Top 1 排名"放右下、把"产品迭代时间线"放左上 → 视线先看到次要信息

## 3. 三分法（rule of thirds）

把页面横竖各分三份，**关键元素放在交点**：

```
┌────┬────┬────┐
│    │  ●│    │
├────┼────┼────┤
│    │    │    │  ●
├────┼────┼────┤
│●   │    │    │
└────┴────┴────┘
```

KPI 大数字、首图主元素、人脸 / 截图主体放在交点处。**不要把单一图表居中铺满**，留空气感。

## 4. 网格（Grid）— 1600×900 的隐性栅格

本系统的隐性栅格：

- **横向**：12 列，gutter 16px，列宽约 117px
- **纵向**：8 行，gutter 16px，行高约 100px
- **slide padding**：48px 顶 / 64px 左右

**所有元素的宽高都是 12 列里的若干列**。`.kpi-grid` = 4×3 列 / `.two-col` = 6+6 列 / `.three-col` = 4+4+4 列 / `.phase-layout` = 3+5+4 列。

**永远不要"目测对齐"**。要么用 grid，要么用 flex `gap`，不要 `position: absolute` + `left: 327px`。

## 5. 留白（Negative Space）

留白不是浪费，是**层次工具**。规则：

- 同组元素 → 小间距（8-12px）
- 不同组元素 → 中间距（16-24px）
- 不同区块 → 大间距（32-48px）

**80% 的"PPT 看起来很乱"问题，根源是间距没有层次**——所有间距都 8px，或者所有间距都 32px。

## 6. 视觉锚点（Anchor）

每页应该有**一个**最显眼的元素抓眼球：

- 总览页 → "5 个" 64px 大数字（col-num）
- 数据页 → KPI 大数字（36px 900 字重）
- 趋势页 → 多色折线图本身
- 故事页 → 一句加粗结论

**只能有一个锚点**。如果两个东西都喊"看我"，就都没人看。次要元素必须降级（颜色 `--text2` 而非 `--text`，字重 500 而非 700）。

## 7. 对齐（Alignment）

- **垂直对齐**：左对齐为主，数字右对齐（等宽字体）
- **水平对齐**：基线对齐 / 顶对齐 / 中线对齐 三选一，**别混用**
- **数字对齐**：`metric-after` 永远右对齐 + `font-family: var(--font-mono)`
- 反例：标题是 800 字重 36px，下面副标题左缩进 4px（眼睛会发现"挪了一下"）

## 7b. 卡片边界对齐（最重要的工程规则之一）

**当一页用多个卡片切分展示区域时，卡片的边界必须严格对齐**——同行卡片顶边/底边一致，同列卡片左/右边一致。这是专业感和混乱感的分水岭。

### 三条硬规则

1. **同行卡片高度一致**：左卡片 8 行 metric、右卡片 6 行 metric，**底边必须齐平**——靠子元素伸缩补偿差异，不让卡片本身变高变矮。
2. **同列卡片宽度一致**：用 grid `1fr 1fr` 或 `repeat(N, 1fr)`，不要 `auto` / 不要按内容撑。
3. **卡片内部留白对齐**：所有卡片用相同 `padding`，内部第一行（标题 / metric / chart-header）的基线在水平方向上要对得上。

### 实现模式

**A. 同行 N 个卡片同高**

```css
.country-row { display: grid; grid-template-columns: 1fr 1fr; gap: 14px;
               flex: 1; min-height: 0; }   /* 父容器要 flex:1 + min-height:0 */
.country-row > .card { display: flex; flex-direction: column;
                       height: 100%; margin: 0; min-height: 0; overflow: hidden; }
```

关键：`grid-template-columns: 1fr 1fr` 让宽度严格相等；父容器 `align-items` 默认 `stretch` → 子卡片自动同高。

**B. 卡片内 N 行 metric 平分高度（撑满到底）**

```css
.country-row > .card > .metric-row { flex: 1 1 0; min-height: 0;
                                     padding: 0; border-bottom: 1px solid var(--border); }
.country-row > .card > .metric-row:last-child { border-bottom: none; }
```

关键：`flex: 1 1 0` 让每行平分剩余高度。**左卡 8 行 / 右卡 6 行时，每行高度不同但底边对齐**。

**C. 异构内容用"弹性占位"补差（phase-arrow 模式）**

左侧是 3 个 phase-box（内容多寡不一），右侧是 2 个等高 chart-card——直接 stretch 会让 phase-box 被拉成超大。解决：

```css
.phase-timeline-v { display: flex; flex-direction: column;
                    min-height: 0; overflow: hidden; }
.phase-timeline-v .phase-box   { flex: 0 0 auto; height: auto; }  /* 自然高 */
.phase-timeline-v .phase-arrow-v { flex: 1 1 0; }                  /* 占位伸缩 */
```

`phase-arrow-v` 是 phase-box 之间的 ↓ 箭头——让箭头吸收剩余空间，**最后一个 phase-box 的底边**就和右侧 chart-card 的底边对齐了。

### `min-height: 0` —— flex/grid 撑满布局的命脉

flex/grid 子项默认 `min-height: auto`，意思是"内容多大我多大"。当父容器要它"高度只能 100%"时，子项内容超出 → 撑爆父容器。**所有 `flex: 1` 或 `align-items: stretch` 的容器，子项都必须显式写 `min-height: 0`**（横向撑满则 `min-width: 0`）。

```css
.parent  { display: flex; flex-direction: column; height: 600px; }
.child   { flex: 1; min-height: 0; overflow: hidden; }   /* ← 缺了就溢出 */
```

90% 的"为什么我的卡片底边对不齐 / chart 莫名其妙撑高 / 滚动条冒出来"问题，根源都是没写这一行。

### 对齐自检（写完 CSS 之后）

- [ ] 同行卡片底边连成一条直线吗？（用浏览器开发者工具拉一根水平线压上去）
- [ ] 同列卡片左/右边连成一条直线吗？
- [ ] 卡片内的"标题"基线水平方向对得上吗？
- [ ] 把窗口拖动 resize，对齐还在吗？（如果某个尺寸下崩，多半是 `min-height:0` 漏了）
- [ ] 内容长度临时改了（例如某 metric-row 删掉一行），其他卡片底边还齐平吗？

### 反模式

- ❌ 用 `height: 380px` 写死卡片高度——窗口缩放或内容变化必崩
- ❌ 用 `margin-top: 12px` 把矮卡片"硬推下去"对齐——内容一改又错位
- ❌ 同行用 `display: inline-block` + `vertical-align: top`——只对齐顶边、底边永远参差
- ❌ 卡片内部 padding 一个 16px 一个 20px——视觉上"差一口气"
- ❌ 漏写 `min-height: 0` → 内容撑高某一卡 → 整行被拉高 → 其他页底边看着齐这页不齐

## 8. 对比（Contrast）

四种对比维度：

| 对比类型 | 实现 | 用途 |
|---|---|---|
| **大小对比** | KPI 36px vs label 13px | 主次 |
| **粗细对比** | 标题 800 vs 正文 400 | 主次 |
| **明暗对比** | text 0f172a vs text3 94a3b8 | 主次 |
| **色彩对比** | 关键数字 accent 蓝 vs 正文 text2 灰 | 焦点 |

**至少用 2 种对比**才能让层次清晰。只用大小对比 → 单调；只用色彩对比 → 看不出主次。

## 9. 重复（Repetition）

视觉一致 = 专业感。同类元素**结构必须重复**：

- 所有 chart-card 的 header 都是 "标题 + 注脚" 两件套，注脚永远在右
- 所有 country-card 都是 "国名 + tag" 头 + N 行 metric-row
- 所有 phase-box 都是 "PHASE NN · 日期" + 标题 + bullet items

**重复带来阅读节奏**。读者第二次看到同结构卡片就不需要重新解析布局，直接看内容。

## 10. 信息密度（Density）

**每页元素数 6-9 个最佳**。少于 4 个 → 留白过多显得空；多于 12 个 → 信息过载。

经验值：

| 页型 | 主体元素数 | 组成示例 |
|---|---|---|
| 总览页 | 8-10 | 3 板块 × 2-5 卡片 |
| KPI + 双对照 | 4 KPI + 2 卡片 = 6 | 顶部指标行 + 双对照表 |
| 三阶段 + 多图 | 3 phase + 4 chart = 7 | 阶段时间线 + 多 chart-card |
| 多对象趋势 | 4 chart = 4 | 多对象 facet |
| 分类 + 趋势 | 4 mini-bar + 1 chart + 1 insight = 6 | 类目排名 + 总趋势 + 解释框 |

**超出 12 个 = 拆成两页**。不要塞。

## 11. 故事曲线（多页之间）

整份 deck 应有节奏，不要 6 页都是同一种密度：

```
高潮：多对象趋势汇总 ━━━━━╲
                            ╲
中等：双对照 ━━━━━━━━━━━━━━━╲━━━ ←—— 信息密度
                              ╲
低（呼吸）：扉页 / 总览 ━━━━━━━━╲
                                ╲
                  ↓ 时间
```

经典节奏：

1. **总览**（低密度，建立框架）
2. **重点 1 / 2 / 3**（高密度，详细数据）
3. **趋势汇总**（中密度，连贯故事）
4. **未来计划 / 风险**（低密度，呼吸 + 收尾）

**两页同样高密度连排** → 读者疲劳。中间插一页低密度（一句话页 / 大数字页 / 全图页）。

## 12. 数据图的 8 条潜规则

1. **轴线弱化**：`axisLine` 用 0.10 alpha 不用纯黑
2. **网格线只留 y 轴**：x 轴的网格线 99% 是噪音
3. **图例放底部**，不要左右——会挤压绘图区
4. **legend / tooltip 字号 ≤ 11px**，给图本身留空间
5. **折线图 ≤ 5 条**，超出用 facet（拆成多个小图）
6. **柱状图柱间距 = 柱宽的 40%**（`barCategoryGap: '40%'`）
7. **里程碑用虚线竖线 + 顶部小标签**，不要画在图里
8. **永远标注数据源 / 取数日期**（chart-card-note 或 footer）

## 13. 反布局清单

立即修正这些：

- ❌ 一页 12 个以上 KPI 数字（拆页）
- ❌ 标题居中、正文左对齐（要么都居中要么都左对齐）
- ❌ 用 `<br>` 控制段落间距（用 margin）
- ❌ 让 chart 填满整张页面没有标题/注脚（用 chart-card 包起来）
- ❌ 关键数字字号比上下文小（要更大、更亮、更粗）
- ❌ 正文大段超过 3 行（拆 bullet 或 callout）
- ❌ 一页里出现 2 种以上的 emoji 国旗 + 文字 emoji 混搭风
- ❌ footer 写"by 张三 2025-04-30"（这是 git commit 信息，不是页面信息）
- ❌ 用渐变色装饰背景（背景必须是纯色或 grid）
- ❌ 让正文字号小于 11px（投屏看不清）

## 14. 自检清单（写完一页问自己）

- [ ] 这页的"一句话结论"是什么？写在 subtitle 里了吗？
- [ ] 关键数字加粗 + 颜色高亮了吗？
- [ ] 视觉锚点是哪个？（应该只有一个）
- [ ] 投屏到 50 米外，最重要的字看得清吗？
- [ ] 把页面打印成黑白，层次还在吗？（仅靠颜色区分主次 = 失败）
- [ ] 删掉一个元素页面会更糟吗？（不会则删）
- [ ] 数据来源标注了吗？
- [ ] 下页是不是同样高密度？要不要加一页"呼吸页"？
